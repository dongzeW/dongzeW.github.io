compile 已被弃用，并由两个新的替代：

implementation：仅仅将你引用到的此 module 在内部使用，而不将其接口暴露给外部

api：通过你自己的接口将你引用到的 module 接口暴露出去供其他 module 使用，含义上实际与 compile 一致

####网络模型(com.sinosig.almightytreasure.net)

#####网络请求整体流程
HttpUtils.java类,该类用于串联被观察者(Observable)和观察者(Observer),把二者放在一个事件流里面。该类可以做网络加载前的一些处理，
比如获取本地存储的数据，做为结果返回。
被观察者由ApiService组成，ApiService.java接口类用于存储所有网络请求接口。
ApiConstants.java接口类，用于存放所有接口请求的地址，包括服务http地址（gradle之前配置过的地址）
Api.java类，用于初始化OkHttp网络请求
观察者由ProgressSubscriber组成，用于处理接口返回的数据
ApiException.java类，结果运行时异常处理类，比如当网络异常时，会回调到该类

#####网络对话框的管理
ProgressSubscriber.java类(观察者)，当调用者Presenter发起网络请求的时候，
会初始化一个progresssubscriber观察者，并将该观察者订阅给被观察者(Observable)，当被观察者的动作完成后，正常会回调到onComplete()
异常会回调到onError()，然后在这两个方法里取消对话框

#####异常的统一处理
ProgressSubscriber.java类，再该类的onError()方法处理，包含ApiException异常,和接口请求异常，该类返回是在RxHelper.java类的
handleResult方法里

#####统一数据格式
BaseResult.java类，定义接口返回的基类,所有接口返回的数据通用模板。数据格式如下

data为接口的真实数据类型,当请求接口的时候在ApiService.java类定义，其中LoginBean对应登录接口返回是data数据

#####自定义Retrofit中的ConverterFactory
当针对不同的数据类型，比如接口返回的数据格式为json或xml时候，我们需要自定义factory，来引入自己的解析方式。
举例，引入阿里巴巴的com.alibaba:fastjson。
自定义实现coverter的实现类，处理responseBody里返回的数据流

将定义好的converter转换器添加到Retrofit中

#####自定义拦截器(Interceptor)
UnsafeOkHttpClient.java类，实现了自定义拦截器，控制网络请求的Log输出，debug级别输出日志，release关闭日志输出

RequestInterceptor.java类，当请求网络接口时候，需要统一设置请求头(header)或请求体(body)时，可以自定义拦截器，进行预处理

####App H5交互(com.sinosig.sunshinelife.utilityplatform.web)

WebView的初始化

定义用户自己的WebView(UIWebView)，用于配置统一处理WebSettings、cookie、封装javascript

添加Javascript接口，调用方法addJavascriptInterface(new NativeWebViewProxy(), “Native”);将NativeWebViewProxy代理类实现的所有方法注入webview，用于h5调用，调用方式如：Native.shareTimeline()，Native为添加方法的时候起的别名,对应addJavascriptInterface方法的第二个参数，shareTimeline()为NativeWebViewProxy()类的实现方法

调用h5url之前设置cookie用于传送(登录信息)数据给h5，

webview设置setWebChromeClient和setWebViewClient方法

WebChromeClient类的主要功能

  1. 用于扩展支持h5的input标签，进行图片的选择和相机的打开，如果h5使用该方式获取图片资源，h5拿到数据后需自己进行图片的压缩处理，否则会因为图片过大，导致不避免的错误发生
  • 添加webview加载页面时的loading页面

WebViewClient类的主要功能

  1. shouldOverrideUrlLoading()方法对sms、tel、mailto等协议的处理
  • onPageFinished()页面加载完成后处理标题数据
